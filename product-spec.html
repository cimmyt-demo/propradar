Product specification (PropFirmMatch-style)

High-level EPICs, user stories and mapped UI surfaces for a prop firm comparison & loyalty platform. This page is meant as a PM/engineering overview, not end-user marketing.

EPIC A — Discovery & Comparison

Core user flows: search all firms, apply structured filters, and compare challenges side-by-side.

  • User story A1: Search all firms with filters for assets, platforms, evaluation type, account size, pricing and restrictions.
  • User story A2: Compare multiple firms side-by-side on challenge rules.
  • User story A3: Category pages (Best Futures Firms, Top Crypto Firms, etc.).

Mapped UI: Firms, Lists, Compare.

EPIC B — Firm Profiles & Challenges

Data model and UI for firm-level pages and challenge catalogs, with structured rule metadata driving search and comparison.

  • User story B1: Firm profile page with rating, years active, country, platforms, assets and trust flags.
  • User story B2: Challenge catalog with detailed rules (targets, drawdowns, time limits, payout schedule).

Mapped UI: conceptual firm detail sections implied across Firms and Compare (ready for a dedicated firm-detail template).

EPIC C — Reviews, Ratings & Trust

Reviews, verified badges and payout/rule-change monitoring to create a trusted layer above simple affiliate listings.

  • User story C1: Submit review with rating, min text length and optional proof upload.
  • User story C2: Verified review badge when purchase is confirmed via proof or affiliate APIs.

Mapped UI: Reviews, trust copy on Overview and future payout/rule-change feeds.

EPIC D — Affiliate Clickout Funnel

Click tracking and attribution used to power both revenue and the loyalty program.

  • User story D1: Redirect flows that attach affiliate IDs and log click events.
  • User story D2: Attribution and reconciliation with affiliate networks or partner APIs.

Mapped UI: “Buy” actions implied on Firms, Compare, Offers and flows described on How it works.

EPIC E — Loyalty Points & Rewards

Points accrual and redemption mechanics layered over the affiliate engine to increase retention and LTV.

  • User story E1: Earn points by uploading order confirmations, with manual or OCR-assisted approval.
  • User story E2: Redeem points against a transparent rewards table for free plans or discounts.

Mapped UI: Loyalty, confirmation flows described in How it works.

EPIC F — Accounts, Favorites & Personalization

User accounts tie together points, favorites, reviews and history, enabling personalization and long-term engagement.

  • User story F1: Profile dashboard with loyalty balance, proofs and reviews.
  • User story F2: Favorites list synced across devices.

Mapped UI: Trader dashboard (concept), plus favorites affordances planned for firm lists.

EPIC G — Offers, Coupons & Promo Engine

Deals hub surfacing coupon codes, time-limited promos and extra-account perks with clear disclosures.

  • User story G1: Offers index and detail model with code, discount, validity and targeting rules.

Mapped UI: Offers, hero badges on Overview.

EPIC H — Admin Backoffice & Partner Dashboards

Internal tools for managing firms, challenges, reviews, loyalty and affiliates, plus a B2B interface for partners.

  • User story H1: Admin panel for firm/challenge CRUD, review moderation, loyalty, offers and analytics.
  • User story H2: Prop firm dashboard with traffic, clicks and performance metrics.

Mapped UI: Prop Firm Business, Admin (concept).

EPIC I & J — Content, Analytics & Anti-Fraud

SEO content surfaces for acquisition, plus analytics and anti-fraud systems to protect users and keep rankings trustworthy.

  • User story I1: Article publishing system with SEO fields and tagging.
  • User story J1: Analytics tracking of searches, clicks, conversions and review health.
  • User story J2: Anti-fraud checks for reviews, order proofs and suspicious patterns.

Mapped UI: Future content hub, analytics tiles on Overview and Prop Firm Business.
